为人民服务的公司：构想与未解问题

版本：v1.0 起草：2026-05-12 状态：早期构想，待打磨

一、核心判断

生产力跃迁正在让"公司"这种生产关系形态变得不合时宜。

软件服务行业正在经历一次类似春秋战国（铁器牛耕）、工业革命（蒸汽机）级别的生产力大爆发——这次的引爆物是 AI 模型能力。当三人小团队加 AI 工具能完成过去三十人团队的工作时，巨头维持的"协调收益"不再值它收取的那部分"地租"。

更深层的判断：AI 时代的"剩余"不会自动以剩余价值的形式被资本吸走，因为生产单位正在变小，资本的协调价值在贬值。如果这个判断成立，那么新的分配机制就不只是道德选择，而是经济必然。

人才是根本。 Token、算力、模型权重，本质上都是生产资料。关键是被谁掌握、为谁服务。这一点跟马克思的核心洞察一脉相承。

二、历史类比

1. 春秋战国类比

铁器牛耕带来农业生产力大爆发 → 旧的井田制秩序崩塌 → 大量剩余人口 → 战争重新分配土地，缓解"人口过剩 vs 农业岗位不足"的矛盾。

2. 工业革命类比

蒸汽机带来工业生产力大爆发 → 旧的农业生产秩序崩溃 → 资本介入，把农业人口赶入工业领域 → 通过压低农产品价格、抬高工业品价格实现资本剥削 → 催生马克思主义。

3. 当下：AI / 软件服务革命

软件服务行业（AI 时代的"第三产业核心"）生产力大爆发 → 旧的互联网公司组织形态成为新型"封建领主"：

  • 腾讯 = 通讯、游戏领主
  • 阿里 = 支付、网购领主
  • 美团 = 本地生活领主
  • 苹果、Google、Meta 同理

封建领主的本质不是"规模大"，而是基于稀缺资源建立的地租提取关系（应用商店、流量入口、社交关系链）。新生产力允许更小的协作单位，意味着这种地租关系需要被重构。

4. 两条已知路径都不该重走

  • 走回宗教指导下的"小国林立"——历史已证明会陷入长期冲突；
  • 走回中国式"大一统"——历史也已证明效率与活力的代价。

有了马克思主义指导，我们应该能找到第三条路。

三、制度设计：为人民服务的公司

3.1 内部代币机制

借鉴比特币的有限发行 + 集体时代的荣誉记账，设计一种不可交易、记名个人的代币：

  • 总量有限：类似比特币的发行曲线，随时间递减；
  • 集体稀释：加入的人越多，单位时间、单位贡献产出的币越少；
  • 非交易性：不能在人与人之间转让，只能由个人凭贡献"挣"，再用来兑换公司内部资源；
  • 使用场景：在公司内部兑换实际生活物品、服务，甚至租房；
  • 获取方式：仅通过对集体的贡献。

3.2 共产度（Communism Degree）

衡量公司"为人民服务"程度的核心参数：

  • 定义：薪酬中代币占比 vs 现金占比的比例；
  • 初始状态：100% 现金 / 0% 代币（与普通公司无异）；
  • 演进机制：通过集体民主投票逐步提高代币比例；
  • 下限保障：现金部分必须不低于当地法定最低工资；
  • 共产度提升的收益：现金支出降低 → 释放资源用于雇佣更多人 / 购买外部服务 / 投入生产。

参考案例：华为的"虚拟股 + 现金"双轨制（注意区别：华为虚拟股可有限度兑现并分红，本设计的代币严格不可交易）。

3.3 外部传染机制

借鉴 GPL 等"传染式"开源协议的思路：

  • 与本公司对接的另一家公司，如果也采用"为人民服务"协议（类似的股权 / 薪资结构），则天然获得更便宜的报价；
  • 报价比例由对方的共产度决定——共产度越高，给到的折扣越大；
  • 形成"为人民服务"网络的正反馈循环，逐步覆盖产业链上下游。

3.4 终极理想形态

每个人的代币账户记录其历史贡献，能换来房子、衣服、车子——但不可交易。这接近马克思设想的共产主义高级阶段（按需分配 + 个人贡献可追溯）。

四、过渡路径

  1. 个体公司层面：从普通私人资本公司起步，逐步提高共产度，进行内部转化；
  2. 公司间网络层面：通过传染式协议，把孤立的"为人民服务公司"连成网络；
  3. 社会层面：从当前中国特色社会主义体制下出发，向这种社会形态过渡。

五、未解问题（待 grill 的清单）

Q1：不可交易代币的灰色市场问题

苏联劳动券、计划经济票证、文革工分制度，都产生过灰色市场。区块链能阻止账面交易，但阻止不了"我帮你刷工分、你把房子借我住"这类现实折价兑换。靠什么机制处理？

Q2：贡献度量问题（兰格-米塞斯之争）

如何客观度量一个程序员 / 产品经理 / 客服 / 行政对集体的贡献？现金薪酬之所以管用（即使不公），是因为市场价格机制聚合了无数主观判断。换成代币以后，谁来定价？投票？那为什么不会重蹈苏联工厂委员会的覆辙（人人给自己高分、给脏活低分）？

Q3：通缩式发币的代际公平问题

2026 年加入的"开创者"和 2030 年加入的"建设者"贡献相同，前者拿到的币会多几倍。这种事实阶层与"共产主义理想"的张力如何化解？

Q4：外部协议的强制执行

GPL 有版权法兜底。"为人民服务协议"靠什么强制执行？如果只是道德契约，假装高共产度骗低价、内部继续剥削的行为如何被识别和制裁？

Q5：与外部资本主义市场的接口

员工的房贷、孩子学费、父母医药费是刚性现金支出。除非公司垂直整合到不可思议的程度（自建食堂、宿舍、医院、学校 = 1980 年前的"单位制"），代币系统如何真正减轻对外部现金的依赖？

Q6：边界问题——谁是"人民"？

  • 仅公司内部成员 → 这是"先进合作社"；
  • 所有交互方（用户、供应商）→ 这是"新型基础设施"；
  • 全社会 → 这是事实上的政党组织。

三种边界对应完全不同的资本结构、决策机制、风险承担。要走哪条？

六、需要进一步夯实的论证

最具价值的不是代币方案本身（其中很多老问题），而是核心判断——"AI 时代协调价值贬值，巨头地租关系不可持续"。

建议补充以下案例论证：

  • 自媒体替代传统媒体公司（协调价值贬值）
  • 独立游戏对大厂（小团队 + 工具链的胜利）
  • AI 编程让独立开发者重新崛起
  • 开源软件的"礼物经济"已经验证了非货币化协作可行

如果这个核心判断能用 3-5 个硬案例夯实，整套构想才有不可绕开的现实根基。

七、参考与可借鉴的历史经验

  • 中国 1949-1978 单位制（功能整合最彻底的现实实验）
  • 中国人民公社（关于度量、激励、退出权的教训）
  • 南斯拉夫工人自治
  • 以色列基布兹（Kibbutz）
  • 西班牙 Mondragon 合作社（最大、最成功的现代合作社联合体）
  • 欧文、傅立叶等空想社会主义实验
  • 华为 TUP / 虚拟股制度
  • 开源运动（GPL、Apache、MIT 三种协议的传染性对比）
  • 比特币 / Web3 DAO（已经踩过的坑很多，特别是"链上治理"的失败）
