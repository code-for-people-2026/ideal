为人民服务的公司:构想 v2.1

版本:v2.1 日期:2026-05-12 对照:v1.0 · v2.0

v2.0 之后的执行层文档——制度骨架 + 法律外壳 + 品牌 + 入伙流程

用途说明:本文档为创始人自用 / 决策清单。不直接给高中同学或外部人看——信息密度和承诺密度过高,需通过分阶段对话呈现(见 §6.2)。

一、v2.0 → v2.1 的关键补充(执行摘要)

v2.0 解决的是"思想框架 + 启动期组织形态"问题。v2.1 不修改 v2.0 的任何核心命题,只在以下层面补充可执行规则和近期行动:

  1. 制度骨架成型——公司治理(BDFL + 数量级触发 + 2/3 决议)、工资 1:3 永恒条款、资金捐款性质、产品决策机制,从模糊承诺压实到可执行规则
  2. 法律外壳路径明确——三阶段路径(90 天试点 → 普通公司 + 章程自约束 → 民非),包括业务主管单位选择、政治敏感性、外汇合规
  3. 中文品牌确定——"为工友敲键盘",法律名 / 品牌名 / 项目认同三层分离
  4. 种子用户成分干预——前 50 用户硬性配额(≥20 来自第一二产业),防止路径锁死为"程序员互助小组"
  5. 入伙流程——学徒已入伙但需确认"满足生活"具体数字;高中同学未入伙,设计四阶段对话
  6. 推迟决策清单形式化——把"现在不决策"的事项明确列出,避免"以后再说"演变为"以后忘了"
V2.1 一句话蒸馏

v2.0 的思想姿态在 v2.1 被压到可执行层——每一项承诺都有具体规则、触发条件、和锁定机制;每一项"以后再决策"都有具体触发条件,不靠记忆维持。

二、制度骨架

2.1 公司治理

Sun 是事实 BDFL(Benevolent Dictator For Life)。这一事实需要书面承认而不是模糊化——"现在大家民主决定"在 3 人小队 + Sun 单一金主的结构下是橱窗,模糊化的代价是当未来某次冲突真的发生,没有任何文件可以指向说"我们当初就是这么约定的"。

规则

数量级触发条件:每扩一个数量级(3 → 30 → 300 → 3000 → 3 万),治理机制必须重新决定,不允许线性渐变。每次跨越数量级,所有规则需要全员重新设计并投票决定。

规则

人事决议:招新人 / 开人 / 批准离职均需 2/3 同意(参照 1918 年新民学会规则)。3 人时即任何两人即可决定,Sun 一票不足以单独决定人事。

2.2 工资约束:1:3 永恒条款

规则·永恒条款

最高工资 ≤ 最低工资 × 3。

应用范围:现金 + 代币 + 任何形式的物质回报。

锁定层级:永恒条款。写入章程的"不可被任何决议修改的条款"清单。即使全员一致同意也不能修改。仿照美国宪法第五条对参议院代表平等的特别保护、德国基本法第七十九条永恒条款机制。

已知代价(feature 而非 bug):

  • 招募池受限——大厂中级程序员年包 50-80 万,1:3 上限下学徒 5500 元/月起、师傅最高约 16500 元/月。这是自动筛选机制——能留下来的要么真信、要么没有别的去处
  • 历史先例:Mondragón 合作社 1956 年成立时即 1:3,1990 年代屈服于外部劳动力市场压力放至 1:6 → 1:9。永恒条款是给未来自己的 commitment device,防止重蹈 Mondragón 路径

2.3 资金性质:只接受捐款

规则

所有进入项目的资金均为捐款,包括创始人投入。

具体含义:

  • Sun 自己投入的房租和工资 → 捐款,不可回收
  • 学徒和高中同学的工资来源 → 项目从 Sun 的捐款支付
  • 未来用户付费 → 项目收入(不属于 Sun)
  • 未来外部捐款 → 直接给项目,捐赠人不获得股权或控制权

关键属性:

  • 创始人投入不可回收。Sun 60 岁离开时,带不走任何东西——这件事注册民非后是法律强制,不是主观选择
  • 项目不可上市、不可并购、不可创始人套现——永远没有 exit
  • 无投资人概念——只有"捐赠人",捐赠不换股权

2.4 BDFL 与决议的边界

决策类型谁决定备注
单产品功能 / bug产品 BDFL(1-3 maintainer 中的带头人)乱搞 → 产品自然挂掉,市场纪律
公司层产品方向选择不存在中央决策,小团队自决"广泛撒网试错",符合宗旨即可开启
招新人 / 开人 / 批准离职2/3 决议新民学会规则
工资比例调整不可决议永恒条款
接受外部捐款公司层 BDFL(Sun)但 BDFL 无任何套现权
治理机制本身的修改数量级触发后全员重新设计不允许渐变

三、法律外壳

3.1 三阶段路径

阶段时间法律形式下阶段触发条件
阶段 1 · 试点0-3 个月不注册任何实体,个人转账 + 个人合同发工资第一个产品有用户 ≥ 100
阶段 2 · 公司3 月 - 用户 ~1000普通有限公司 + 章程自约束(不分红、利润全转工资、解散归社会)找到业务主管单位 + 第一阶段稳定运营满 6 个月
阶段 3 · 民非用户 1000+社会服务机构(民办非企业)—

阶段 2 是关键过渡期:享受有限公司便利(开发票、签合同、招人),通过章程约束实现"民非的精神"。这一段是法律意义上"Sun 个人控股"的灰色地带,需 Sun 在心理上把这段时间也当作"暂时托管"。

3.2 形式选择:为什么民非

形式适用性主要原因
公募基金会❌注册资金 ≥ 800 万
非公募基金会⚠ 边缘注册资金 ≥ 200 万,定位是"资助方"非"运营方"
社会团体❌适合行业协会,不适合产品型项目
民非(社会服务机构)✅5-30 万注册资金,可收服务费,可发工资,禁止分红,资产归社会

民非的关键吸引点:

  • 无股权,无所有权——跟"创始人投入也算捐款"完全一致
  • 可收服务费——B2C 软件订阅收入合法
  • 可发工资——1:3 工资比可正常执行
  • 禁止分红——盈余必须再投入业务,法律级保证"无 exit"
  • 解散后资产归社会——比章程锁死强,连理论可能性都消除

3.3 业务主管单位与政治敏感性

民非注册的真实门槛不是注册资金,是业务主管单位(政府某个部门承担监管责任)。

候选(按当前判断的可能性排序):

  1. 民政局——部分省市试点直接登记,可跳过业务主管单位(优先尝试)
  2. 教育部门 / 人社部门——若框架成"职业培训"
  3. 共青团——若框架成"青年技能教育"
  4. 网信办 / 工信部——若框架成"数字素养"
  5. 总工会 / 地方工会——最对口但对独立工友组织态度复杂,放在最后试
政治敏感性

中国"工友 / 劳工"主题 NGO 自 2015 年起处于持续紧缩环境。番禺打工族服务部、深圳手牵手等多数已停止运作。这是结构性环境,不可改变。

实务结论:法律名 ≠ 品牌名。法律注册用"数字素养 / 灵活就业服务 / 信息技术普惠"等漂白名称,品牌叙述保留"为工友敲键盘"。里子和面子分开是合法的、必要的、不算虚伪。

3.4 外汇合规

Sun 资金来源包括外企美元工资。操作路径决定一切——同样的钱走不同路径,法律意义完全不同。

规则·安全路径

美元 → 境外个人账户 → 入境结汇 → 境内人民币账户 → 完税 → 以人民币、个人名义捐赠给项目实体。

账面记录:"个人捐赠,捐赠人 Sun,人民币"。与外汇/外企/境外完全无关。

禁止操作
  • 外企直接以美元打款给项目
  • 境外账户直接为项目支出(再以"报销"形式回款)
  • 接受境外组织资金或品牌授权
  • 在境外注册相关组织再支持境内实体

任何一条都会实质性踩到《境外 NGO 管理法》。

实操细节(本周对一遍):

  • 个人结汇额度 5 万美元/年。超过需走"个人特殊业务",备齐工资单 + 完税证明 + 用途证明
  • 完整保留外企工资条 + 完税证明,5 年内可能被询问
  • 所有项目支出从境内项目账户出,不在境外为项目消费
  • 避免每年 3 月、7 月、10 月等敏感时点的大额结汇 / 捐赠

四、品牌与传播

4.1 中文品牌:为工友敲键盘

选这个名字的理由:

  • "工友"——既指服务对象,也是程序员自嘲("码农"、"互联网工友"在 996.ICU 之后已成真实自称)。涵盖第一二产业劳动者 + 中产程序员学徒的双重人民
  • "敲键盘"——把"编程"这个略带精英气的词降到工友能听懂的层级;同时带"键盘侠"的自嘲味,消解"为工友"可能引发的居高临下感
  • 整体语气是劳动姿态:"我们不是什么科技公司,我们就是一群敲键盘的"

简称:"工友敲键盘" / "敲键盘"用于非正式场合。

商标提示

"为工友敲键盘"作为使用不需审批,可马上启用。但注册商标大概率被驳回——描述性过强(《商标法》第 11 条缺显问题)。补救路径是"为工友敲键盘 + 臆造词后缀"或"图形 + 文字"组合。商标注册可推迟到项目规模有起色后再处理。

4.2 公司法律名分层

层级名称用途
法律名(公司 / 民非)XX 数字素养服务中心(或类似漂白名)工商登记、合同、税务、对接政府部门
品牌名(对外)为工友敲键盘产品、官网、自媒体、用户口语
项目认同(对内)为人民服务的公司创始团队内部、师徒誓约、本档文件系列

三层分离是设计选择,不是虚伪——法律外壳与精神核心解耦,是中国监管环境下的现实必要。

4.3 自媒体定位

核心:吸引目标用户,而不是技术分享(避免引来程序员观众池)。

内容方向:

  • AI 工具教学(目标受众:第一二产业劳动者)
  • 工友处境观察 + 解决思路
  • 项目过程透明分享(失败也是内容)
算法层面的现实预期

抖音 / B 站 / 小红书的推荐系统按"内容标签 + 用户画像"做相似性推荐。"工友 AI 教学"会被推给目标受众,也会推给"对底层叙事感兴趣的中产观众"——后者点赞但不转化为用户。

预期落差:6 个月内涨粉 5 万但转化用户 500。这是结构性差,不是做得不够好。

4.4 获客方法

核心:到群众中去。绑定既有线下基础设施,比自建社群效率高 10 倍:

  • 工友驿站(外卖员歇脚点,全国几万个)
  • 工地党群服务中心
  • 县城技工学校 / 中职
  • 工会职工书屋
  • "工会户外劳动者驿站"等

90 天试点期内即可启动一线行动,不需要等任何治理设计完成。

五、种子用户与扩散

5.1 前 50 用户成分干预

决定一个软件社区基因的不是规模 1000 之后的多数人,是 50 以内的活跃少数。

规则

前 50 个种子用户中,硬性 ≥ 20 个来自第一二产业。

不付这个代价的后果:产品早期从程序员圈传播开,种子用户 80% 是程序员,一年后第一二产业用户进入会被"程序员氛围"挤出。半年后"为工友敲键盘"自动退化为"程序员互助小组"。

具体代价(必须承担):

  • 前期获客效率显著下降——程序员通过 demo 链接就来了,工人需要面对面、需要被理解他们的真实需求
  • 需要主动到工友驿站、技校等地方做线下接触,这是 90 天试点期的核心活动之一
  • 产品体验对工人可能还不完善,但要硬接住前 20 个,在他们身上迭代

5.2 学徒裂变机制(承袭 v2.0)

出师后回乡开类似公司,师承网络扩散。

v2.1 新增的现实判断:学徒出师后大概率不会复制本项目的全部承诺(包括 1:3 永恒条款、民非形式、外汇合规等)。师承裂变不要求基因 100% 复制,精神方向延续即足够——他/她在自己的地方开普通商业公司服务工友,也是项目目标的一部分。

六、人员入伙流程

6.1 学徒(已入伙)

当前位置:南通大学软件工程专业毕业,已自费就近租房投入项目实战,愿意接受半薪试用 750 USD / 月。

自我定位原话:

"我觉得像马克思那样穷困潦倒、颠沛流离我做不到,但是像恩格斯那样在满足生活的情况下搞革命是可以的。"

这是极其清醒的自我边界——他不是"含糊愿意奉献",是"精确知道愿意奉献到什么程度"。

需要 Sun 心里标记的三件事:

  1. 他是"参与者"不是"共承诺者"。困难期不能假设他陪走超出"满足生活"线之下的部分
  2. "满足生活"的具体数字必须 7 天内确认——他现在住哪、月固定支出多少、5500 元 / 月够不够。如果不够,要么调数字、要么调住地。这个错配会在第二个月就显化
  3. 出师后大概率回乡开普通商业公司,可能做工友相关产品但用传统公司形式。这件事 OK,不要试图强求他复制全部承诺

6.2 高中同学(待入伙·关键执行)

当前位置:未提及完整构想。本职 3000 USD / 月稳定远程,中级前端程序员。

已知信息:

  • Sun 是他的技术 mentor(把他带进前端行业)——已存在师徒关系
  • 价值观与 Sun 接近,对马列主义和各类哲学的学习可能比 Sun 更多
  • 35 岁左右,可能有家庭
关键风险
  1. 现有师徒关系会让他难以说"不"。出于义气可能加入,但未必是 visceral 认同。义气入伙是炸弹,半年后必爆
  2. 理论比 Sun 深,他的 grill 力度会比预期大。需准备好被反向 grill,不要因为是 BDFL 就硬撑维护自己的版本——他能 grill 是项目的资产,不是威胁

分阶段对话(必须遵守,不可一次性把本文档给他看)

阶段内容听他什么
第一次产品想法 + "为工友"宗旨有没有 visceral 共鸣?
第二次师徒制 + 工资 1:3 + 90 天试点接受 3000 → 1500 USD/月差价?
第三次公司治理(BDFL + 数量级触发 + 2/3 决议)接受 Sun 做 BDFL?他想要什么权力?
第四次
(可能 1-2 月后)		长期承诺(民非 + 无 exit + 不可回收)最终承诺测试

关键执行细节

  • 第一次见面面对面,不视频通话、不文字、不电话
  • 第一次明说一句话:"无论你最后是否加入,我们的关系不会因此改变,我不希望你因为'欠我'而加入,我希望你因为'你也想做'而加入" —— 把"退出空间"从理论变成事实,让他听见
  • 允许每阶段退出。如果他在第二阶段说"工资差价 1500 USD 接受不了"——损失是两顿饭,重新找人。如果直接灌完本文档让他签字,他在心里默默退场,半年后才显化——损失是 6 个月精力 + 一个朋友
  • 本文档对自己用,不直接给他。它是 Sun 在每次对话前对照的备忘录,而不是给他读的"已设计完毕的合同"

七、推迟决策清单

明确列出当前阶段不决策的事项,防止"以后再说"演变为"以后忘了"。每项都标记触发条件——一旦满足,就必须真正决策,不能继续推。

7.1 议程权与投票机制

触发条件:用户规模 ≥ 500 OR 活跃贡献者 ≥ 50

当前选项库存(不实际选择):

  • 双轨制:产品功能 BDFL + 组织事务投票
  • 流动民主 + 委托上限 10% + 委托衰减 90 天
  • AI 代理民主:已被否决——会沿阶级线复制不平等

7.2 代币系统

触发条件:第一个产品有稳定收入 + 内部经济活动复杂到需要中间货币

v2.0 时空着,v2.1 仍空着。需要回答:

  • 代币的功能(投票 / 消费 / 收益分配 / 都有?)
  • 发行机制(贡献奖励? 出资换取?)
  • 1:3 上限应用于"个人持有量"还是"年度新增分配"?
  • 衰减机制是否需要

7.3 多团队资源分配

触发条件:9-12 个月后,如果"广泛撒网试错"真的产生 ≥ 3 个并行小团队

当前选项库存:中央分配 / 自筹捐款 / 成熟产品养实验产品。三种各有代价,推迟到压力出现再选。

7.4 反向师徒的具体机制

触发条件:第一个产品有稳定用户社区

v2.0 提到,v2.1 未压实。何同学失败模式警告:单向抽取不署名不分钱 ≠ 反向师徒。需回答:工人学员的知识如何进入产品、决策、收入分配,不只是当素材库。

7.5 解散程序

触发条件:阶段 3(民非)注册前

民非法律规定解散后资产归社会,但"归社会"的具体路径需在章程中预先指定。比如"指定捐赠给某某基金会或同类公益机构"。这件事写在章程里,创始人对未来失败有更诚实的预期。

八、不可解决问题清单(更新)

v2.0 原 6 项保持不变(见 v2.0 文档第五节)。v2.1 新增 3 项:

7. 法律形式过渡期 = 政治时间漂移期:从公司转民非需要 1-2 年。这段时间项目实际形态是"个人控股公司",与"创始人不掌握所有权"的承诺有名义上的不一致。代价是 Sun 需要在心理上把这段时间也当作"暂时托管",不当作"我的资产"——这件事承诺易、心理难。

9. "满足生活"标准会随地域和家庭责任浮动:学徒(刚毕业、无家庭)和高中同学(35 岁、可能有家庭)对"满足生活"的具体数字差距很大。1:3 永恒条款在年轻无家庭者身上 OK,在有家庭责任的合伙人身上可能不够——这是"信仰者过滤"的副产品。预案是接受高中同学如果因家庭经济压力退出,不视为背叛,关系不变。
