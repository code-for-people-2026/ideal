# 为人民服务的公司

> 对外主题：**数据平权、AI 下乡** ｜ 对外品牌：**码成工 —— 一个为工友敲键盘的组织**（Code for People）
>
> 软件也是一种服务，为何不还给人民。

一支正在筹备的小型软件团队（3 人）的构想档案与第一次产品。

## 我们是谁

我们想在 AI 时代，为普罗大众——我们称之为"工友"——做软件，并把软件这种服务，更多地还给使用它、生产数据的人。

"工友"不按行业，按人在生产关系中的位置划分：不占有平台、数据与算法，因而不得不向其交租、出卖劳动来维持生活的一切人——一产二产劳动者、服务业新蓝领、个体经营者、失业待业者、无偿再生产劳动者（全职妈妈、照顾者）、被去技能化的低级白领。

## 名字的寓意：码成工

英文名 **Code for People**，承"为人民服务"一脉，简洁有力。中文名 **码成工**，三字，是一句藏在成语里的纲领。

它从"马到成功"化出：**马 → 码**（code），**功 → 工**（同音 gōng，工友/工人），省去"到"字，留三字成名。读音仍近"马（到）成功"，意思却全是我们的旗号。一名两读，两读都成立：

- **以码成工** —— 用代码成全工友、把软件这种服务还给人民。这是**宗旨**：我们为谁。
- **码（农）成工** —— 写代码的人自己也是工人。在 AI 让软件能力白菜化、码农被去技能化的今天，这是一句**阶级站位**的宣告：我们是谁。

它与宗旨标语 **"码上为民"**（"码上"谐"马上"）互为表里——一个亮主体（码成工），一个亮宗旨（码上为民）。

> 标准署名（名 + 注脚）：**码成工 —— 一个为工友敲键盘的组织**。"码成工"是对懂行人发的暗号，"为工友敲键盘"是对谁都讲得通的白话注脚；旧直译名由此降级为副标题，不丢、也不抢戏。

> 命名沿革（诚实条款）：中文名先后为直译的"为工友敲键盘"，2026-06 定为"码成工"。早期头脑风暴文档（`初期头脑风暴/`）保留旧名作为思想演化的存档，不回溯改写。

## 我们的判断

过去二十年，互联网平台坐在生产者与消费者中间收租，活成了"数字封建领主"。但 AI 让写软件的能力白菜化，算力与软件产能开始过剩。

> 稀缺的时代，胜负看谁能生产；过剩的时代，胜负看红利让给谁。

## 我们主张四件事

1. **数据属于用户** —— 平台盈利向全体用户分红：看的人、创作上传的人都分，不只激励头部 up 主。这就是"数据平权"。
2. **AI 下乡** —— 仿"家电下乡"。当年与今天，背景同样是过剩；把过剩的算力，用去深挖普罗大众的真实需求。
3. **普及 AI 教育，防范 AI 资本** —— 别让 AI 重蹈土地、工业、金融、互联网资本被少数人窃取的覆辙。
4. **收入差距 < 3:1** —— 团队内部现金收入与可支配收入，差距都不超过三倍，作为分配示范。

## 现场互动页部署

这个仓库里的互动矩阵是 Next.js + Payload CMS。摊位现场二维码必须指向公网部署地址，不能指向 `localhost`。

### Vercel 环境变量

至少需要在 Vercel 配置：

- `PAYLOAD_DATABASE_URL` 或 `DATABASE_URL`
- `DATABASE_URL_UNPOOLED`，如果数据库供应商提供
- `PAYLOAD_DATABASE_SCHEMA=ideal_interactive`
- `PAYLOAD_SECRET`
- `SUBMISSION_IP_HASH_SALT`
- `NEXT_PUBLIC_SITE_URL`
- `ADMIN_EMAIL`
- `ADMIN_PASSWORD`

### 首次初始化线上数据库

```bash
vercel login
pnpm prod:init
```

### 重新生成二维码

```bash
NEXT_PUBLIC_SITE_URL=https://你的线上域名 pnpm qr:flyer
```
