为人民服务的公司：构想 v2.0

版本：v2.0 日期：2026-05-12 对照：v1.0 →

经过 9 轮辩证 grilling 后的修正版本

一、v1.0 → v2.0 的关键转变（执行摘要）

v1.0 是一个思想姿态级别的构想：从马克思主义 + 比特币 + 集体记忆的混合出发，提出"代币 + 共产度 + 民主投票"的乌托邦企业形态。论证强项在历史类比和理论框架，弱项在经济可行性、组织动力学、阶级实现三块。

v2.0 是一个可启动级别的构想。经过 9 轮针对性 grilling，主要修正：

  1. 承认"代币 + 民主"是终极愿景而非启动机制——启动期是师徒制 + 创始人主导，明确为 v3.0 的演化目标；
  2. 从"内部乌托邦"转向"外部服务运动"——接受列宁式 vanguard 框架，中产学徒（内圈）+ 工人学员（外圈）两层结构，呼应 1922 年安源工人夜校的组织形态；
  3. 从 B2B 软件转向 B2C 直接服务——拒绝雇主作为客户，避免成为"数字鞭子"，依托 AI 让小规模软件可行的新经济学；
  4. 从"代币系统"转向"模型主权 + 集体议价工具"——重新校准"生产资料"概念：数据是原料，模型 + 算力 + 训练能力才是生产资料；
  5. 从"教育"扩展到"教育 + 互助 + 组织"三条腿——拒绝纯慈善型 AI 素养教育，互助内置于产品，组织以 voice 形态出现（集体定价、统一品牌、对工厂/医院的议价工具）。

二、当前构想的核心命题

  1. 生产力跃迁判断：AI 让"最优软件生产单位"变小。三人小团队 + AI 工具可以做过去三十人产品团队做的事，让"专为 500 人做一个软件"从不可行变成可行。
  2. 服务对象：第一/二/服务业劳动者（外卖骑手、仓库工人、家政、服务员、小工厂工人），以及他们的家庭。
  3. 不直接转化的服务方式：不试图把工人变成程序员（不科学），而是增加他们对 AI 的可及性、提供他们处境改善所需的工具。
  4. vanguard + mass base：内圈是中产理想主义学徒（全职），外圈是工人学员（副业革命）。所有人都同时是学习者，包括创始人本人。
  5. 三条腿不可分：教育（AI 素养课程）+ 互助（产品内置的社区互助网络）+ 组织（集体议价、统一品牌、互联网形态的工会替代物）。
  6. 技术主权方向：短期国内廉价 API + DeepSeek 等开源模型为主，长期 fine-tuning + 自部署。避免成为大平台 AI 基础设施的依附性下游。
  7. 裂变式扩散：学徒"出师"后回到自己熟悉的地方，开一家类似的公司。靠师承网络扩散，不靠融资 + 雇佣模式。
V2.0 一句话蒸馏

一个由中产理想主义者主导的"软件师徒会社"，通过自有模型 + 自有算力，给被零工经济裹挟的工人提供 AI 素养教育、互助网络、和集体议价工具——让工人在 AI 时代不只是更高效率的被剥削对象，最终通过出师裂变扩散为分布式的"为人民服务"网络。

三、9 轮 grilling 演化轨迹

ROUND 1不可交易代币 → 灰色市场问题
攻击

苏联劳动券、文革工分、计划经济票证都产生了无法阻止的灰色市场。代币只要能换实物，就会形成私下交易渠道（代购、人情债、家庭账户合并、批条权力寻租）。

修正

接受流通本身不可怕，可怕的是"既享受使用价值，又身居高位享有投票权"。引入"代币 = 投票权 + 消费券双重凭证"的机制——花掉代币会降低投票权。

ROUND 2外部财富购买内部权力
攻击

投票权与持有代币挂钩的设计，结构性地让有外部财富的人保留全部投票权（不需消耗代币），而真正需要保障的人消耗代币导致投票权衰减。这是资本主义结构的镜像复刻。基布兹（Kibbutz）就是被这个问题侵蚀的——能留下来的越来越是中产以上家庭。

修正

引入两个机制——(a) 投票本身也消耗代币（每张票一次性）；(b) 累积代币随时间衰减。让"治理权"从存量变成流量，无法囤积。

ROUND 3内部劳动 vs 外部使命的异化
攻击

第二轮的修正导致一个新立场——"为人民服务 = 为外部人民服务，内部成员只是手段"。这是任何资本主义公司的同构结构（劳动者 + 工资 → 服务 → 顾客）。马克思把这叫异化，你只是把"被资本异化"换成了"被使命异化"。天主教传教士修会、Wikipedia、开源维护者、Patagonia 都是这个失败模式的案例。

修正

重新定义"人民"——内外无差别。接受列宁式 vanguard 框架，承认现在与共产主义之间有巨大差距，需要过渡桥梁，桥梁本身有张力但是必要的。

ROUND 4B2B 软件 = 数字鞭子（digital whip）
攻击

声称"为工人优化生产"的 B2B 软件，实际是卖给管理者用来更密集监控 + 算法节拍 + 去技能化 + 替换工人的工具。Amazon 仓库、美团骑手、客服坐席都是历史教训。马克思自己用《资本论》大量篇幅证明"技术不改变生产关系就不会带来工人解放"。

修正

转向 B2C，直接为消费端服务——给依法受雇者提供生活软件（家政、药品、婴儿护理），给自由职业者提供工作软件。论据是 AI 让小规模软件经济可行（long tail software）。

ROUND 5工人付不起 → 钱从哪来
攻击

你想服务的人定义上是支付能力最弱的人。第一/二/服务业劳动者月入 3500-8000，可承担软件订阅每月不超过 30 元——而且这 30 元已经被腾讯/字节分掉。如果产品免费，钱从哪来？广告？补贴？每条路都带来治理俘获。

修正

自筹启动（远程工作月薪 10w 支持 3-4 人小团队） + 中产家庭应届生为主 + 鼓励成员保留外部工作。拒绝传统"先算账后开干"思路，采用 Customer Development（搭讪 + 24h demo + 三三制小组）。

ROUND 6创始人 = 单一金主 → 隐性独裁
攻击

你是唯一真实经济基础（远程工作工资 100%），所谓"民主决策"在你掌握现金流的情况下是橱窗。中国民营企业有一个成熟的失败剧本——"理想主义启动 → 创始人个人帝国 → 理想被现金流稀释"。

修正

承认"代币 + 民主"是终极形态，启动期是师徒制。已经有一个学徒自费在 Sun 家附近租房学编程。设计"双重出师"：个人级出师是技术独立 + 自养能力，公司级出师是回到熟悉的地方开一家类似公司，靠师承裂变扩散。

ROUND 7自费学徒制 = 阶级过滤器
攻击

自费学徒结构系统性排除工人阶级子弟（外卖骑手家的孩子、农村第一代大学生、单亲家庭青年）。你的招募事实上是中产家庭过滤器。这正是 20 世纪所有马列实践失败的核心机制——党核心成员的阶级背景与声称代表的群体不重合，"代表性"变成修辞。

修正

参考 1922 年安源工人夜校——"副业革命，主业管生计"的两层结构。内圈是中产学徒（cadres），外圈是工人学员（mass base）。明确放弃"把所有工人变成程序员"的乌托邦，目标降到"增加 AI 可及性"。批判碎片化视频（抖音/小红书）作为学习来源，主张系统性课程。

ROUND 8教育没有互助和组织 = 慈善而非革命
攻击

安源工人夜校之所以有效，是三条腿结构（教育 + 工人消费合作社/医疗互助 + 工人俱乐部→工会→大罢工）。你只有教育这一条腿——产出的觉悟工人无处可去，最有能力的工人会用 AI 技能跳出工人阶级（"技能培训的政治保守性"问题）。

修正

互助内置于软件社区（母婴 app 例子：二手流转 + 工厂集体议价 + 医疗互助 + 互相带娃）。组织以"私域流量小平台"形态出现（网约车司机绕过滴滴接私单的现象）。明确判断传统工会（美国走狗化、中国福利化）已过时。

ROUND 9数据 ≠ 生产资料 / 化整为零 = exit 而非 voice
攻击 1（范畴错误）

数据是原材料（19 世纪类比：棉花），不是生产资料（纺织机）。19 世纪农民拥有自己种的棉花仍被低价收购，因为没有纺织机。AI 时代真正的生产资料是模型 + 算力 + 训练能力。

攻击 2（exit vs voice）

网约车司机绕过滴滴接私单是"逃避剥削，不是消除剥削"。Hirschman 的 Exit / Voice 分析：exit 削弱 voice，最不满的人离开导致剩下的人更顺从。

修正

技术主权目标重新校准为模型主权——短期国内廉价 API + DeepSeek 开源，长期 fine-tuning + 自部署。创始人承认自己目前没有这个能力，需要在研发中学习。集体定价、互助保险、统一品牌、对工厂/医院的集体议价工具被接受为产品方向——把化整为零从 exit 升级成 voice。

五、不可解决问题清单（诚实声明）

把这些写在文档里，是为了让自己和后续加入的人都不会忘记这些是当前阶段无法消除的张力，不是设计漏洞。

1. 创始人 = 单一金主：代币 + 民主机制在此结构下物理上无法真正运转。明确承认前 2-3 年是创始人开明专制期，不要包装成民主。

2. 学徒招募 = 阶级过滤器：自费学徒结构系统性排除工人阶级子弟。组织内部成员构成与服务对象不重合。通过"工人学员（外圈）副业革命"模式缓解但不消除。

3. 教育抽走最优秀工人：技能培训项目历史上的政治保守性问题（最有能力的工人通过学习离开本阶级）。只能通过"出师创业 + 留下扎根本地"导向缓解。

4. AI 基础设施暂时仍依附：短期使用国内 API，创始人本人目前没有 fine-tuning 能力，需要在研发中学习。2026-2027 年仍然是依赖期。

5. 化整为零的政治意义有限：私域流量绕开大平台是 exit 而非 voice。承诺把它升级为 voice，但工具能否真的产生集体力量需要时间验证。

6. 小规模软件经济可行性是赌博：100+ 方向探索 + 自媒体兜底是一个有道理但风险高的策略——3 人小队 100 方向 = 平均每方向 3% 注意力。

六、应变机制（contingency）

学徒在出师前想退出

当前方案签带违约费用的合同。

注意与"自愿性 + 知情性"的 vanguard 原则有内在张力。建议合同条款明确"违约费用上限 = 培训期实际成本，不超过工资总额"，避免演变为强制性指证劳动。否则在道德论证上跟邪教合同区分不开。

产品 PMF 验证失败

当前方案多方向同时探索（争取 100 个方向） + 同时进行自媒体创业（失败也转化为流量）。

注意3 人小队 100 方向 = 每方向 3% 注意力，过于分散。建议演化为"3-5 个主方向并行 + 每周 1 个新方向轻量探索 + 自媒体作为常态化副产品（每个方向的过程本身就是内容）"。自媒体作为兜底是合理设计——它把"失败的项目"转化为"成功的内容"。

Sun 的 10w/月远程工作突然没了

当前方案两份远程工作互为备份（同时被开除概率低） + 大哥罩着（被开除也有补偿） + 已形成的财务缓冲。

评估这是 v2.0 里结构最稳健的一块。备份冗余 + 社会资本 + 财务缓冲三重保护，是真实可行的风险管理。比一份工作"梭哈"的创业者结构性更安全。

八、待启动的下一步

  • 90 天执行规划：第一个 MVP 产品方向、用户研究路径、学徒培养节奏、自媒体矩阵筹划
  • AI 自部署 PoC：选一个开源模型（DeepSeek-V3 / Qwen 等），完成第一次 fine-tuning 实验
