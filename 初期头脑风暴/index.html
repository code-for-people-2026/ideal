为人民服务的公司

A.I. 时代新型企业组织形态的设计档案

已知，软件服务是一种服务。
为人民服务是一种崇高的使命。
现在 AI 模型能力爆发，导致软件服务生产力大爆发。
生产过剩，需要消费升级。

这是一个长期演化的思想 + 实践项目。我们处在类似春秋战国（铁器牛耕）、工业革命（蒸汽机）级别的生产力大爆发——这次的引爆物是 AI。互联网巨头作为"封建领主"的形态正在失效。我们需要在马克思主义指导下，找到第三条路：既不是宗教指导下的小国林立，也不是大一统。

V1.0 · 思想姿态

初始构想：代币 + 共产度 + 民主投票

2026-05-12 起草。从马克思主义 + 比特币 + 集体记忆的混合出发，提出乌托邦企业形态。论证强项在历史类比和理论框架，弱项在经济可行性、组织动力学、阶级实现三块。

 V2.0 · 可启动级别

修正版：师徒制 + 工人 AI 夜校 + 模型主权

经过 9 轮辩证 grilling 后的可启动版本。承认"代币 + 民主"是终极愿景；启动期采用列宁式 vanguard（中产学徒）+ mass base（工人学员）结构；从 B2B 转向 B2C；技术主权从依赖 API 走向开源模型自部署；教育 + 互助 + 组织三条腿不可分。

 V2.1 · 执行层就绪

制度骨架 + 法律外壳 + 品牌 + 入伙流程

v2.0 之后的执行层文档。压实公司治理（BDFL + 数量级触发 + 2/3 决议）、工资 1:3 永恒条款、资金捐款性质；明确法律三阶段路径（试点→公司→民非）；确定中文品牌"为工友敲键盘"；设计高中同学的四阶段入伙对话。对自己用，不直接给外部人看。

 附 · 90 天执行规划

田野 + 技术 + 上线 + 合规的 13 周展开

v2.1 第 9 节"本周行动"的展开版。包含选题方法论、田野调查具体安排（注册骑手 / 应聘电子厂 / 妇幼场景替代方案）、技术架构、上线路径选择、ICP 备案、数据保护合规、13 周周历、决策点与失败应变。三人对照执行用。
关于三个版本的关系：v1.0 是思想起点，v2.0 是被 9 轮 grilling 压到地面之后的可启动版本，v2.1 是把 v2.0 的承诺压到具体可执行规则 + 近期行动的执行层文档。三个版本对照阅读，能看到一个构想如何从"思想姿态"走到"可启动级别"再走到"执行层就绪"——每一步推进都是诚实回应锐利反驳的结果。
